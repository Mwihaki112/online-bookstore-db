### Part1 questions

1. What is a database relationship? This defines how tables are connected to each other using keys. It allows data in one table to relate to data in another, reducing duplication and improving organization.

2. Explain:
- One-to-One (1:1): Each record in one table is linked to only one record in another table.

- One-to-Many (1:M): One record in a table can be linked to many records in another table.

- Many-to-Many (M:N): Many records in one table relate to many records in another table. This requires a junction (bridge) table.

3. What is a foreign key? A foreign key is a column in one table that references the primary key in another table. It creates a relationship between the two tables.

4. Give one real-life example of each relationship type
- One-to-One: One person has one passport
- One-to-Many: One customer can have many orders
- Many-to-Many: A student can take many courses, and a course can have many students
